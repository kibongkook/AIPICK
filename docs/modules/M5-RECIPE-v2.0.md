# M5-RECIPE v2.0: AI 레시피 — 멀티옵션 레시피 시스템

> **버전**: 2.0 (멀티옵션 확장)
> **상태**: `ai-song-creation` 1건 전환 완료, 점진적 확장 중
> **인덱스**: [M5-RECIPE.md](./M5-RECIPE.md) | 이전 버전: [M5-RECIPE-v1.0.md](./M5-RECIPE-v1.0.md)
>
> **핵심 콘셉트**: 하나의 목표, 다양한 경로.
> 완전 초보부터 고급 사용자까지, 같은 결과물을 만드는 여러 방법을 단계적으로 안내한다.

---

## 1. 현재 시스템의 한계

### 현재 구조 (v1)
```
AIRecipe
├── difficulty: 'easy' | 'medium' | 'hard'  ← 레시피 전체에 하나의 난이도
├── steps: RecipeStep[]                       ← 하나의 고정 경로
│   └── alt_tools?: string[]                  ← 대안 도구(이름만 나열)
└── (pros/cons 없음)
```

### 문제점
| 문제 | 설명 |
|------|------|
| **단일 경로** | 하나의 레시피 = 하나의 방법. "다른 방법은 없나?"에 답할 수 없음 |
| **alt_tools가 피상적** | 대안 도구를 나열만 하고, 왜/어떻게 쓰는지 안내 없음 |
| **난이도 고정** | 초보자에겐 어렵고, 고급자에겐 새로울 게 없는 "중간" 레시피만 존재 |
| **장단점 부재** | 어떤 방법이 더 빠른지, 더 퀄리티 높은지 비교 불가 |
| **성장 경로 없음** | 쉬운 방법에서 시작해 고급 방법으로 넘어가는 흐름이 없음 |

## 2. 새로운 콘셉트: 멀티옵션 레시피

### 핵심 원칙

```
하나의 레시피 = 하나의 목표 + 여러 옵션(경로)
각 옵션 = 독립된 난이도 + 단계 + 장단점 + 추천 대상
옵션은 쉬운 것부터 어려운 순으로 정렬
```

### 사용자 여정 시나리오

```
[초보 사용자 "민수"]
  "AI로 노래 만들고 싶은데 뭘 어떻게 해야 하지?"
     ↓
  레시피 진입: "AI로 나만의 노래 만들기"
     ↓
  옵션 1(아주 쉬움) 발견: "Suno에서 바로 만들기"
     ↓
  3분 만에 첫 노래 완성! 성공 경험
     ↓
  "더 내 맘대로 만들고 싶다"
     ↓
  옵션 2(쉬움) 도전: "GPT로 가사 쓰고 Suno에서 생성"
     ↓
  더 만족스러운 결과물!
     ↓
  옵션 3(보통) 도전: "가사+편곡+마스터링 풀 워크플로우"
     ↓
  프로 수준 결과물 달성
```

## 3. 데이터 구조 설계

### 3-1. 새로운 타입 정의

```typescript
// ==========================================
// AI 레시피 v2 — 멀티옵션 시스템
// ==========================================

/** 난이도: 5단계로 확장 */
export type RecipeDifficultyV2 =
  | 'very-easy'   // 아주 쉬움 — 클릭 몇 번으로 완성
  | 'easy'        // 쉬움 — 기본 프롬프트 입력 수준
  | 'medium'      // 보통 — 여러 도구 조합
  | 'hard'        // 어려움 — 세밀한 설정 + 반복 작업
  | 'expert';     // 전문가 — 외부 도구 연동 + 고급 기법

/** 하나의 단계 (v1과 동일, 확장) */
export interface RecipeStepV2 {
  step: number;
  title: string;
  tool_slug: string;
  tool_name: string;
  action: string;
  prompt_example?: string;
  tip?: string;
  /** 이 단계의 예상 소요 시간 */
  estimated_time?: string;     // "1분", "5~10분" 등
  /** 선택적 단계 여부 (건너뛰어도 됨) */
  optional?: boolean;
}

/** 옵션 내 하나의 경로 */
export interface RecipeOption {
  /** 옵션 식별자 */
  option_id: string;           // "direct", "gpt-suno", "full-workflow" 등
  /** 옵션 제목 */
  title: string;               // "Suno에서 바로 만들기"
  /** 한 줄 설명 */
  subtitle: string;            // "프롬프트 한 줄이면 3분 안에 노래 완성"
  /** 이 옵션의 난이도 */
  difficulty: RecipeDifficultyV2;
  /** 이 옵션의 예상 총 소요 시간 */
  estimated_time: string;
  /** 사용하는 도구 수 */
  tool_count: number;
  /** 이 옵션의 단계들 */
  steps: RecipeStepV2[];
  /** 장점 목록 */
  pros: string[];
  /** 단점 목록 */
  cons: string[];
  /** 이 옵션을 추천하는 대상 */
  best_for: string;            // "처음 AI를 접하는 완전 초보자"
  /** 결과물 퀄리티 (5점 만점) */
  quality_rating: 1 | 2 | 3 | 4 | 5;
  /** 자유도/커스텀 가능성 (5점 만점) */
  customization_rating: 1 | 2 | 3 | 4 | 5;
}

/** AI 레시피 v2 */
export interface AIRecipeV2 {
  slug: string;
  title: string;
  subtitle: string;
  category: RecipeCategory;    // 기존 카테고리 재사용
  /** 레시피 전체의 대표 난이도 (가장 쉬운 옵션 기준) */
  difficulty_range: {
    min: RecipeDifficultyV2;
    max: RecipeDifficultyV2;
  };
  /** 최종 결과물 설명 */
  result_description: string;
  tags: string[];
  icon: string;
  color: string;
  /** 옵션 목록 (쉬운 순 → 어려운 순 정렬) */
  options: RecipeOption[];
  /** 옵션 비교 요약표 (선택) */
  comparison_summary?: string;
}
```

### 3-2. v1 → v2 호환성

```typescript
/** v1 레시피를 v2로 변환하는 어댑터 */
function migrateRecipeV1toV2(v1: AIRecipe): AIRecipeV2 {
  return {
    ...v1,
    difficulty_range: { min: v1.difficulty, max: v1.difficulty },
    options: [{
      option_id: 'default',
      title: v1.title,
      subtitle: v1.subtitle,
      difficulty: v1.difficulty,
      estimated_time: v1.estimated_time,
      tool_count: v1.tool_count,
      steps: v1.steps.map(s => ({ ...s })),
      pros: [],
      cons: [],
      best_for: '',
      quality_rating: 3,
      customization_rating: 3,
    }],
  };
}
```

## 4. 완성 예시: "AI로 나만의 노래 만들기"

```
slug: "ai-song-creation"

result_description: "가사, 멜로디, 보컬이 포함된 완성된 노래 (+ 앨범 아트, 뮤직비디오)"
icon: "Music"
```

### 옵션 1: Suno에서 바로 만들기 (아주 쉬움)

| 항목 | 값 |
|------|-----|
| **난이도** | 아주 쉬움 (very-easy) |
| **소요 시간** | 3~5분 |
| **사용 도구** | Suno AI (1개) |
| **퀄리티** | ★★☆☆☆ (2/5) |
| **자유도** | ★☆☆☆☆ (1/5) |
| **결과물** | MP3 파일 |
| **추천 대상** | AI를 처음 써보는 완전 초보자 |

**단계:**

| # | 제목 | 도구 | 하는 일 |
|---|------|------|---------|
| 1 | Suno 접속 & 주제 입력 | Suno AI | suno.com 접속 → Create → 자연어로 주제 입력 → Create 클릭 |
| 2 | 결과 선택 & 다운로드 | Suno AI | 2개 결과 중 선택 → MP3 다운로드 |

**장점:** 3분 완성, 진입 장벽 제로, 무료 하루 10곡
**단점:** 가사 직접 작성 불가, 곡 구조 제어 불가, 결과 랜덤

---

### 옵션 2: AI 작사 + Suno 생성 (쉬움)

> ChatGPT **또는** Gemini 중 편한 도구 선택 사용

| 항목 | 값 |
|------|-----|
| **난이도** | 쉬움 (easy) |
| **소요 시간** | 15~30분 |
| **사용 도구** | ChatGPT 또는 Gemini + Suno AI (2개) |
| **퀄리티** | ★★★★☆ (4/5) |
| **자유도** | ★★★★☆ (4/5) |
| **결과물** | 내 가사가 담긴 MP3 파일 |
| **추천 대상** | 가사를 직접 컨트롤하고 싶은 사용자 |

**단계:**

| # | 제목 | 도구 | 하는 일 |
|---|------|------|---------|
| 1 | AI로 가사 작성 | ChatGPT/Gemini | 장르·분위기·주제 → 가사 생성 ([Verse], [Chorus] 태그 포함) |
| 2 | 스타일 프롬프트 생성 | ChatGPT/Gemini | Suno용 영어 스타일 묘사 (아티스트명 금지, 사운드 특징으로) |
| 3 | Suno에서 음악 생성 | Suno AI | Custom Mode → 가사 + 스타일 붙여넣기 → 생성 |

**ChatGPT 강점:** 한국어 가사 자연스러움
**Gemini 강점:** 음악 이론(코드 진행, BPM, 키) 제안, 참고곡 파일 첨부 분석

**장점:** 가사 100% 제어, 곡 구조 정밀 제어, 모두 무료
**단점:** 시간 3~5배 소요, 프롬프트 학습 필요

### 옵션 3: 노래 + 앨범 아트 제작 (보통)

| 항목 | 값 |
|------|-----|

| **퀄리티** | ★★★★☆ (4/5) |

| **추천 대상** | 음원 플랫폼 업로드 or SNS 공유를 제대로 하고 싶은 사용자 |

**단계:**

| # | 제목 | 도구 | 하는 일 |
|---|------|------|---------|
| 1 | 가사 + 스타일 + 비주얼 컨셉 | ChatGPT/Gemini | 가사 + Suno 프롬프트 + 앨범 아트 컨셉을 한 번에 생성 |

**장점:** 노래+커버 패키지로 완성도 대폭 상승, 전문적 외관, 디자인 능력 불필요
**단점:** 이미지 생성 도구 추가 학습, 비주얼 컨셉 구상 필요

### 옵션 4: 풀 프로덕션 + 뮤직비디오 (어려움)

| 항목 | 값 |
|------|-----|
| **난이도** | 어려움 (hard) |
| **소요 시간** | 2~4시간 |
| **사용 도구** | ChatGPT/Gemini + Suno + DALL·E/Imagen + Runway + CapCut (5개) |
| **퀄리티** | ★★★★★ (5/5) |
| **자유도** | ★★★★★ (5/5) |

| **추천 대상** | YouTube 채널 운영자, AI 크리에이티브 프로젝트 도전자 |

**단계:**

| # | 제목 | 도구 | 하는 일 |
|---|------|------|---------|
| 1 | 가사 + 스타일 + 뮤비 기획 | ChatGPT/Gemini | 가사 + Suno 프롬프트 + 앨범 아트 + 뮤비 스토리보드(6~8장면) |
| 2 | Suno에서 음악 생성 | Suno AI | 곡 생성 + 파트별 타임스탬프 메모 |
| 3 | 앨범 커버 아트 생성 | DALL·E/Imagen | 1:1 커버 + 16:9 YouTube 썸네일 |
| 4 | AI 뮤직비디오 영상 생성 | Runway Gen-4 | 장면별 5~10초 클립 생성 (6~8장면) |
| 5 | 영상 편집 & 최종 조립 | CapCut | 음악 + 영상 클립 + 자막 + 전환 효과 → 완성 |

**장점:** 완전한 음악 프로젝트 패키지, YouTube 바로 업로드 가능, 카메라/스튜디오 불필요
**단점:** 5개 도구 사용, Runway 무료 크레딧 제한적, 영상 편집 기초 필요

## 6. UI/UX 설계

## 7. 난이도 5단계 기준

| 난이도 | 라벨 | 기준 | 예시 |
|--------|------|------|------|
| `very-easy` | 아주 쉬움 | 하나의 도구에 한 줄 입력. 클릭 몇 번으로 완성 | Suno에 "슬픈 팝송" 입력 |
| `easy` | 쉬움 | 1~2개 도구. 기본 프롬프트 작성 능력 필요 | GPT로 가사 → Suno 생성 |
| `medium` | 보통 | 2~3개 도구 조합. 각 도구별 설정 조정 | GPT+Midjourney+CapCut 조합 영상 |
| `hard` | 어려움 | 3개 이상 도구. 세밀한 설정 + 반복 작업. 전문 지식 일부 필요 | 캐릭터 일관성 + 다중 장면 생성 |
| `expert` | 전문가 | 외부 도구 연동, API 활용, 자동화 파이프라인 | Make.com 자동화 + API 연동 |

## 8. 레시피 작성 가이드라인

### 8-1. 옵션 설계 원칙

1. **최소 2개, 최대 5개 옵션** — 너무 적으면 의미 없고, 너무 많으면 선택 장애
2. **반드시 "아주 쉬움" 옵션 포함** — 완전 초보가 3분 안에 성공 경험을 할 수 있어야 함
3. **난이도는 오름차순** — 옵션 1이 가장 쉽고, 마지막이 가장 어려움
4. **각 옵션은 독립적** — 옵션 1을 하지 않아도 옵션 3을 바로 할 수 있어야 함
5. **결과물은 동일 카테고리** — 모든 옵션이 "같은 종류의 결과물"을 만듬 (노래, 영상, 글 등)

### 8-2. 옵션별 필수 포함 요소

| 요소 | 필수 여부 | 설명 |
|------|-----------|------|
| title | 필수 | 옵션을 한 줄로 설명하는 제목 |
| subtitle | 필수 | 왜 이 방법을 선택해야 하는지 |
| difficulty | 필수 | 5단계 중 택 1 |
| estimated_time | 필수 | 현실적인 소요 시간 |
| steps | 필수 | 최소 1단계, 각 단계에 프롬프트 예시 포함 |
| pros | 필수 | 최소 2개 장점 |
| cons | 필수 | 최소 1개 단점 (정직하게) |
| best_for | 필수 | 이 옵션을 선택해야 하는 구체적 대상 |
| quality_rating | 필수 | 1~5 정수 |
| customization_rating | 필수 | 1~5 정수 |

### 8-3. 프롬프트 작성 원칙

- **복사-붙여넣기 가능해야 함** — 사용자가 그대로 써도 동작하는 프롬프트
- **변수 부분은 명확히 표시** — `[여기에 주제 입력]` 형식
- **한국어 + 영어 병기** — 도구가 영어만 지원하면 영어 프롬프트도 제공
- **결과 예시 언급** — "이렇게 입력하면 이런 결과가 나옵니다" 설명

### 8-4. 장단점 작성 원칙

- **장점은 구체적으로** — "좋다" (X) → "무료로 하루 10곡 생성 가능" (O)
- **단점은 정직하게** — 단점을 숨기면 신뢰 하락
- **비교 관점으로** — "옵션 1보다 퀄리티 2배" 같은 상대 비교 포함
- **해결 가능한 단점에는 팁 추가** — "단점: 영어만 지원 → 팁: GPT로 번역하면 OK"

## 부록 A: 옵션 설계 체크리스트

- [ ] 각 옵션의 장단점이 최소 2개/1개 이상 있는가?
- [ ] 각 옵션의 프롬프트를 복사-붙여넣기만 하면 동작하는가?
- [ ] 옵션 간 난이도가 오름차순인가?
- [ ] 모든 옵션이 같은 종류의 결과물을 만드는가?
- [ ] best_for가 구체적인가? (X: "모든 사람" / O: "처음 AI를 접하는 완전 초보자")
- [ ] quality_rating과 customization_rating이 현실적인가?
- [ ] 성장 경로 안내가 포함되어 있는가? ("이 다음엔 옵션 N을 시도해보세요")
- [ ] 모든 tool_slug가 AIPICK 데이터베이스에 존재하는가?

## 부록 B: 자주 쓰는 옵션 패턴

### 패턴 1: "직접 생성" → "프롬프트 강화" → "멀티 도구"

```
옵션 1: 전문 도구에 바로 입력 (very-easy)
옵션 2: GPT/Claude로 프롬프트 최적화 후 전문 도구 (easy)
옵션 3: 여러 도구 조합 풀 파이프라인 (medium~hard)
```

적용: 이미지 생성, 음악 생성, 영상 생성, 로고 디자인

### 패턴 2: "단순 생성" → "구조화" → "전문화"

```
옵션 1: AI에 한 줄 요청 (very-easy)
옵션 2: 개요/구조를 먼저 잡고 섹션별 생성 (easy)
옵션 3: 리서치 + 생성 + 편집 + 최적화 (medium~hard)
```

적용: 블로그, 이메일, 보고서, 프레젠테이션

### 패턴 3: "템플릿 활용" → "커스텀 생성" → "처음부터 직접"

```
옵션 1: AI 템플릿/프리셋 사용 (very-easy)
옵션 2: 템플릿 기반 + 커스텀 수정 (easy)
옵션 3: 백지에서 시작, 완전 커스텀 (medium)
```

적용: 웹사이트, SNS 콘텐츠, 이력서, 포트폴리오
