## 핵심 질문 3가지에 대한 분석

### Q1. AI 레시피를 보고 따라하는 사람이 정말 있을까?

**결론: 거의 없다. 현재 레시피 모델은 근본적으로 재설계 필요.**

#### 왜 안 따라하는가

```
현재 레시피 구조 (예: "AI로 나만의 노래 만들기")
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
STEP 1 → ChatGPT에서 가사 생성    → 탭 전환, 로그인
STEP 2 → Suno에서 작곡           → 새 서비스 가입, UI 학습
STEP 3 → 결과물 다운로드          → 포맷 변환
STEP 4 → Audacity에서 편집       → 프로그램 설치, 학습
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

사용자 이탈 지점:
STEP 1  ████████████████████  100% 시작
STEP 2  ██████████            50% (새 서비스 가입에서 이탈)
STEP 3  ████                  20%
STEP 4  ██                    10% (완주)
```

**이탈 원인 3가지:**

| 원인 | 설명 |
|------|------|
| **서비스 전환 마찰** | 각 스텝마다 다른 사이트로 이동, 회원가입, UI 학습 필요. 요리 레시피와 다름 — 요리는 같은 주방에서 하지만 AI 레시피는 매번 다른 주방으로 이사해야 함 |
| **즉시 보상 부재** | 4단계를 다 해야 결과물이 나옴. 초보자는 1단계에서 "와 이것만으로도 충분하다"고 느끼고 멈춤 |
| **난이도 착시** | "쉬움 ★★☆"이라 써있어도 초보자에게 새 서비스 3개를 쓰는 건 절대 쉽지 않음 |

#### 레시피 전환 방향

**현재**: "여러 도구를 조합하는 워크플로우" (파워유저용)
**변경**: "하나의 도구로 즉시 결과를 얻는 가이드" (초보자용)

```
AS-IS (현재 레시피)                    TO-BE (변경 방향)
━━━━━━━━━━━━━━━━━━━                   ━━━━━━━━━━━━━━━━━━━
4~5단계, 도구 3~4개                    1~2단계, 도구 1개
결과물까지 15~30분                     결과물까지 1~3분
"따라하세요"                           "이 프롬프트 복사 → 붙여넣기 → 끝"
완주율 10%                             완주율 60%+
```

**핵심 변화: 레시피의 역할 전환**

| | 현재 | 변경 후 |
|---|------|--------|
| **1차 목표** | 워크플로우 교육 | 즉시 결과물 체험 |
| **구조** | 긴 튜토리얼 | 복사→붙여넣기 원샷 |
| **성공 기준** | 전체 완주 | 첫 결과물 확인 |
| **멀티 도구** | 기본 | 선택 (레벨업 후) |

### Q2. AIPICK에 오는 사람은 누구인가?

**결론: 3개 층위가 있고, 현재 AIPICK은 가장 큰 층위(초보자)에 최적화되어 있지 않다.**

#### 사용자 피라미드

```
                    ▲
                   /  \
                  / 5% \     ③ "더 잘 뽑고 싶은 사람"
                 /      \    → 이미 몇 개 써봄, 더 좋은 결과물 원함
                /────────\   → 프롬프트 최적화, 도구 비교, 새 도구 발견
               /   15%    \
              /            \ ② "써보긴 했는데 모르는 사람"
             /──────────────\→ ChatGPT 써봤음, 다른 건 모름
            /     80%        \→ "이미지도 AI로 만들 수 있어?"
           /                  \
          /    ① "AI 뭐야?"    \→ 이름만 들어봄, 안 써봄
         /                      \→ "ChatGPT가 뭔데?"
        /────────────────────────\
```

#### 각 층위의 니즈 vs 현재 AIPICK

| 층위 | 니즈 | 현재 AIPICK | 문제 |
|------|------|------------|------|
| ① 완전 초보 (80%) | "나한테 필요한 AI가 뭐야?" | 295개 도구 목록, 12개 카테고리 | **압도당함.** 295개 중에 뭘 골라야? |
| ② 가벼운 경험자 (15%) | "ChatGPT 말고 다른 것도?" | 카테고리/랭킹/비교 | OK, 하지만 "왜 이걸 써야 하는지" 부족 |
| ③ 결과물 추구자 (5%) | "더 좋은 결과물 뽑는 법" | 레시피, 커뮤니티 | 레시피 구조가 너무 무거움 |

#### 현재 AIPICK의 근본 문제

**초보자가 홈페이지에 들어오면 보는 것:**

```
현재 홈페이지
┌────────────────────────────────────────┐
│ "295개 AI 서비스 분석 중"               │  ← 숫자만 보임, 의미 없음
│                                        │
│ [커뮤니티 통계 배너]                    │  ← 초보자에겐 관심 없음
│                                        │
│ [오늘의 PICK / 급상승 / 당신에게 맞는]   │  ← "나에게 맞는"은 좋지만
│                                        │     로그인 안 한 초보자에겐 랜덤
│ [AI 레시피 캐러셀]                      │  ← 레시피가 뭔지도 모름
└────────────────────────────────────────┘

초보자 반응: "뭘 해야 하지?" → 이탈
```

**초보자가 보고 싶은 것:**

```
"나는 _____ 하고 싶어"
→ "이거 쓰세요" (1개)
→ "이렇게 입력하세요" (프롬프트 1개)
→ "이런 결과가 나와요" (예시)
→ 끝
```

### Q3. 초보자 → 단골로 성장시키는 구조

**이것이 AIPICK의 진짜 비즈니스 모델이다.**

#### 사용자 성장 단계 설계

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

LV.1 "첫 만남"                    ← 80%가 여기
│
│  진입: "유튜브 썸네일 만들고 싶어" (검색/SNS/입소문)
│  경험: 도구 1개 + 프롬프트 1개 → 결과물 즉시 확인
│  감정: "와, 이게 돼?" (놀라움)
│  AIPICK이 해야 할 일: 즉시 결과물 체험시키기
│
│  ✅ 성공 조건: 3분 안에 첫 결과물 봄
│  📌 이 순간 북마크/가입 유도
│
├──────────────────────────────────────────
│
LV.2 "관심 확장"                  ← 재방문 시작
│
│  진입: "저번에 이미지 만들었는데 글쓰기 AI도 있어?"
│  경험: 다른 카테고리 도구 발견, 2~3개 도구 경험
│  감정: "이것도 되네, 저것도 되네" (탐색 쾌감)
│  AIPICK이 해야 할 일: "이것도 해보세요" 추천
│
│  ✅ 성공 조건: 2번째 카테고리 도구 사용
│  📌 개인화 추천 시작 (사용 이력 기반)
│
├──────────────────────────────────────────
│
LV.3 "품질 추구"                  ← 단골 전환 지점
│
│  진입: "Midjourney가 Leonardo보다 나은가?"
│  경험: 같은 목적 도구 비교, 프롬프트 최적화
│  감정: "더 잘 뽑고 싶다" (성장 욕구)
│  AIPICK이 해야 할 일: 비교, 프롬프트 팁, 커뮤니티
│
│  ✅ 성공 조건: 도구 비교 + 프롬프트 개선 경험
│  📌 커뮤니티 참여 유도 (결과물 공유)
│
├──────────────────────────────────────────
│
LV.4 "조합 활용"                  ← 핵심 리텐션 유저
│
│  진입: "이미지 + 텍스트 + 영상을 조합하고 싶어"
│  경험: 멀티 도구 레시피 활용 (이때 비로소 복합 레시피 의미 있음)
│  감정: "나 꽤 잘하는데?" (자신감)
│  AIPICK이 해야 할 일: 고급 레시피 + 커뮤니티 기여 유도
│
│  ✅ 성공 조건: 복합 레시피 완주
│  📌 레시피 제작자로 전환 (UGC)

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

#### 현재 AIPICK vs 필요한 AIPICK

```
현재 AIPICK은 LV.3~4를 위해 설계됨
실제 사용자의 80%는 LV.1

       현재 AIPICK이 타겟하는 곳
                    ↓
LV.1  ████████████████████████  80%  ← 여기를 못 잡고 있음
LV.2  ██████                    15%
LV.3  ███                        4%  ← 여기를 위해 만들어짐
LV.4  █                          1%
```

## 구체적 변경 사항

### A. 홈페이지 — 완전히 재설계

| 현재 | 변경 |
|------|------|
| "295개 AI 서비스 분석 중" | **"뭘 하고 싶으세요?"** (자연어 입력창) |
| 커뮤니티 통계 배너 | **인기 작업 버튼** ("썸네일 만들기", "보고서 쓰기", "배경 제거"...) |
| 로테이션 배너 (오늘의 PICK) | **Before→After 갤러리** (AI로 이런 게 가능해요) |
| 레시피 캐러셀 | **"3분 안에 해보기"** 즉시 체험 카드 |

```
변경된 홈페이지 구조
┌────────────────────────────────────────────────┐
│                                                │
│         "AI로 뭘 하고 싶으세요?"                 │
│         [____________________________] 🔍       │
│                                                │
│  인기:  [썸네일 만들기] [글 요약] [배경 제거]      │
│         [번역] [코드 작성] [로고 만들기]           │
│                                                │
├────────────────────────────────────────────────┤
│                                                │
│  ⚡ 3분 안에 해보기 (즉시 체험)                   │
│                                                │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐       │
│  │ 📸       │ │ ✍️       │ │ 🎵       │       │
│  │ AI 이미지 │ │ AI 글쓰기 │ │ AI 음악  │       │
│  │ 생성      │ │          │ │ 만들기   │       │
│  │          │ │          │ │          │       │
│  │ "고양이   │ │ "자기소개 │ │ "신나는  │       │
│  │  우주복"  │ │  서 써줘" │ │  팝송"   │       │
│  │          │ │          │ │          │       │
│  │ [해보기→] │ │ [해보기→] │ │ [해보기→] │       │
│  └──────────┘ └──────────┘ └──────────┘       │
│                                                │
├────────────────────────────────────────────────┤
│                                                │
│  🔥 "이런 것도 AI로 돼요?"                      │
│  [Before→After 갤러리 슬라이드]                  │
│                                                │
├────────────────────────────────────────────────┤
│                                                │
│  📊 이번 주 가장 많이 찾은 AI                    │
│  1. ChatGPT — 만능 대화형 AI                    │
│  2. Midjourney — 최고 품질 이미지 생성            │
│  3. Suno — AI 음악 작곡                         │
│                                                │
└────────────────────────────────────────────────┘
```

### B. 레시피 → "퀵 가이드" 체계로 전환

| 레벨 | 이름 | 구조 | 대상 |
|------|------|------|------|
| **퀵** | 원샷 가이드 | 도구 1개 + 프롬프트 1개 → 결과 | LV.1 초보 |
| **스탠다드** | 기본 레시피 | 도구 1~2개, 2~3단계 | LV.2 가벼운 경험자 |
| **프로** | 워크플로우 | 도구 3개+, 4~5단계 | LV.3~4 추구자 |

현재 200개+ 레시피 중 대부분이 "프로" 레벨. "퀵" 레벨이 거의 없음.

### C. 검색/탐색 — 도구 중심 → 작업 중심 전환

```
현재: 카테고리 = "이미지·디자인", "글쓰기·문서", "코딩·개발"
      → 도구의 기능별 분류 (도구 관점)

변경: 카테고리 = "SNS 콘텐츠 만들기", "업무 자동화", "학교 과제"
      → 사용자의 할 일 기반 분류 (사용자 관점)
```

| 현재 (도구 관점) | 변경 (사용자 관점) |
|-----------------|-------------------|
| 이미지·디자인 (42개 도구) | 썸네일/배너 만들기, 로고 만들기, 사진 보정, 캐릭터 그리기 |
| 글쓰기·문서 (38개 도구) | 보고서 쓰기, 블로그 쓰기, 이메일 쓰기, 자기소개서 쓰기 |
| 영상·콘텐츠 (25개 도구) | 유튜브 영상 만들기, 릴스 만들기, 영상 자막 달기 |

### D. 도구 상세페이지 — "이걸로 뭘 할 수 있는지"를 최우선으로

```
현재 도구 상세페이지 구조        변경 방향
━━━━━━━━━━━━━━━━━━━━━        ━━━━━━━━━━━━━
1. 도구 설명                  1. "이걸로 이런 게 가능해요" (결과물 갤러리)
2. 레이더 차트 (점수)           2. "지금 바로 해보기" (플레이그라운드)
3. 무료 사용량                 3. 무료 사용량 + "무료로 충분한가?"
4. 기능 평가                   4. "이 프롬프트 따라해보세요" (퀵 가이드 3개)
5. 댓글                       5. 커뮤니티 결과물
6. 유사 서비스                 6. "더 좋은 대안" (비교)
```

### E. 플레이그라운드 — 레시피 안에 숨겨두지 말고 전면 배치

현재 플레이그라운드는 RecipeStepCard 안에 있어서 레시피를 열어야만 쓸 수 있음.
이건 LV.3~4 유저만 도달함.

**변경**: 홈페이지, 도구 상세페이지, 검색 결과 등 모든 곳에서 즉시 실행 가능하게.

### F. 사용자 성장 트래킹 (신규)

현재 AIPICK에 없는 기능:

```
┌──────────────────────────────────────────┐
│ 🏅 나의 AI 활용 레벨                      │
│                                          │
│ LV.2 — AI 탐색 중                        │
│ ████████████░░░░░░░░  60%               │
│                                          │
│ ✅ 첫 AI 결과물 만들기                    │
│ ✅ 2가지 카테고리 사용                     │
│ ☐ 프롬프트 최적화 시도                    │
│ ☐ 커뮤니티에 결과물 공유                   │
│                                          │
│ 다음 미션: "이미지 AI 프롬프트 개선하기"    │
│ → 보상: 플레이그라운드 추가 실행 +2회       │
└──────────────────────────────────────────┘
```

| 2 | **레시피** | 4~5단계 복합 워크플로우만 200개 | 원샷 퀵가이드 추가 (전체의 70%) | **최우선** |
| 3 | **플레이그라운드** | 레시피 스텝 안에 숨겨져 있음 | 홈/상세/검색 등 전면 노출 | **최우선** |
| 4 | **카테고리** | 도구 기능별 (이미지, 텍스트...) | 사용자 작업별 (썸네일, 보고서...) | 높음 |
| 5 | **도구 상세** | 점수/차트 → 기능 나열 | 결과물 갤러리 → 즉시 체험 → 퀵가이드 | 높음 |
| 6 | **성장 시스템** | 없음 | 레벨/미션/보상 (재방문 루프) | 중간 |
| 7 | **온보딩** | 없음 (바로 목록 페이지) | 3문항 퀴즈 → 맞춤 첫 도구 추천 | 중간 |
